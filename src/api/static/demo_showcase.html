项目离线演示页

这不是重新跑 worker 的实时演示，而是把已经批改完成的本地任务组织成一条稳定的展示动线。你明天只需要启动 FastAPI，让页面从现有 SQLite 中读取结果即可，不依赖重新上传、模型外呼或 Redis 状态流是否可用。

任务创建向导 任务进度页 班级结果看板 报告展示页 复核工作台

怎么使用这页

先启动 API：uvicorn src.main:app --host 127.0.0.1 --port 8000。
然后打开 /demo-showcase，按下方按钮依次进入真实的任务进度页、班级看板、报告页和复核台。
这次演示不需要重新发起批改任务；数据直接来自本地已有的 outputs\grading_database.db。

任务进度页

这一页对应“任务已经提交后，老师最关心有没有在正常推进、还要等多久、结果能不能看”。

班级看板

这一页不再是一份份翻结果，而是先看到班级整体水平、主要错因和值得优先关注的样本。

单份报告

进入某个学生后，系统会把扣分点、证据片段和修改建议组织成老师能直接看的报告，而不是原始 JSON。

复核工作台

如果样本复杂或证据偏弱，系统会把风险样本集中起来，老师只需要重点看最值得看的那一部分。

定位

说“教师侧 AI 阅卷与复核工作台”，不要说“全学科、全场景教学平台”。

边界

说“适合单题、小批量、单机试点”，不要说“已经可以直接大规模云服务”。

价值

强调“批量提效 + 风险样本复核 + 讲评依据生成”，不要强调“完全替代老师”。
