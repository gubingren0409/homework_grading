## 🎯 任务目标

将“标准答案”和“学生作答”统一纳入同一条布局预处理管线，确保认知层接收到的是同粒度、可对齐的切片输入，避免非对称输入导致的注意力失焦与误判。

本阶段核心原则：
- 前端数据特征差异（印刷体/手写体）由 **Prompt 策略** 吸收；
- 不通过“架构分叉”处理差异；
- 管线保持单一路径、低熵、可维护。

---

## 一、为什么必须这样做

### 1）认知契约必须严格对齐

认知层本质是对比推理。若一侧是整页、另一侧是切片，会造成上下文噪声，显著提升幻觉风险。

因此必须保证：
- `REFERENCE` 与 `STUDENT_ANSWER` 都走布局提取 + 物理切片；
- 最终输入认知层时具备可比性与可对齐性。

### 2）阻断架构熵增

若分别维护两套物理裁切逻辑（标准答案一套、学生作答一套），状态机复杂度会快速膨胀并违反 SRP。

正确做法：
- 单一 Agent 接口；
- 通过 `context_type` 做策略路由；
- 输出同构 `LayoutIR` 契约。

## 二、已实现内容（代码证据）

## 1）统一预处理入口（编排层）

文件：`src\orchestration\workflow.py`

已实现方法：`_layout_preprocess(image_bytes_list, context_type)`

逻辑：
1. 调 `extract_layout(...)` 获取 `LayoutIR`  
2. 调 `slice_image_by_layout(...)` 做物理裁切  
3. 异步持久化切片  
4. 若切片为空，回退原图

对称接线：
- `run_pipeline(...)` 使用 `context_type="STUDENT_ANSWER"`
- `generate_rubric_pipeline(...)` 使用 `context_type="REFERENCE"`

这两处接线是“对称设计成立”的关键证据。

## 2）Prompt 级多态（感知层）

文件：`src\perception\engines\qwen_engine.py`

- 已实现 `extract_layout(..., context_type, target_question_no, page_index)`
- 将 `context_type` 注入用户提示词
- 复用同一底层 JSON 请求能力 `_call_qwen_json(...)`

文件：`src\perception\prompts.py`

- `LAYOUT_ANALYSIS_PROMPT` 契约含：
  - `"context_type": "REFERENCE" | "STUDENT_ANSWER"`

## 3）统一数学裁切模型（工具层）

文件：`src\utils\image_slicer.py`

- `slice_image_by_layout(image_bytes, layout, output_format="PNG")`
- 统一处理流程：
  - 坐标夹紧到 `[0,1]`
  - 单调性修正
  - float→int 边界防护
  - 最小非空裁切防护

结论：裁切逻辑与语义无关，不区分“印刷体/手写体”。

## 4）契约约束加固（Schema 层）

文件：`src\schemas\perception_ir.py`

- `LayoutIR.context_type: Literal["REFERENCE", "STUDENT_ANSWER"]`
- `image_width/image_height` 由宿主注入，不信任模型原始回传
- `regions` 使用归一化 bbox 契约
