# 🚀 AI 批改系统项目进度与技术细节总结报告 (Phase 26.5)

## 1. 项目定位与核心架构
本系统是一个针对**理科（物理/数学）**自动批改的高可靠异步流水线。
- **感知层 (Perception - VLM)**: 使用 Qwen-VL-Plus/Max，负责将手写试卷图像转化为结构化的 **Perception IR (JSON)**。
- **认知层 (Cognition - LLM)**: 使用 DeepSeek-Reasoner (R1)，负责逻辑校验、分步评分及生成个性化评语。
- **编排层 (Orchestration)**: `GradingWorkflow` 负责图像预处理、并行感知调用、多页 IR 聚合、短路逻辑及认知评估的串联。

## 2. 关键技术突破与最新特性
### 🛡️ 鲁棒性与稳定性 (Reliability)
- **不对称降级路由 (Asymmetric Degradation)**: 
  - **故障模式**: 针对 Reasoner (R1) 存在的流式传输不稳定性 (`incomplete chunked read`) 或长尾样本死循环问题。
  - **对策**: 引入 `MAX_CONNECTION_ERRORS = 1` 激进阈值。一旦触发，自动降级至 `deepseek-chat` (V3)，并强制关闭 `stream`，改用全量同步请求，确保任务必得合法 JSON。
- **熔断 Key 池 (Circuit Breaker KeyPool)**: 实现多 API Key 轮询与 429 状态自动熔断，支持高并发环境下的自动伸缩。

### 🧠 智能评判算法 (Cognitive Alignment)
- **抗位移评判**: 强制要求模型基于感知层 Key 独立比对。即使学生漏答中间题目，后续空位依然能精准判分，消除“连坐扣分”。
- **OCR 容错机制**: 提示词级注入语义等价逻辑（如 $\alpha$ 与 a 等视觉识别误差不扣分）。
- **空白卷短路**: 感知层识别到 `is_blank` 后，编排层物理切断后续认知算力消耗，直接返回 0 分报告。

### ⚡ 性能优化 (Efficiency)
- **异步全流程**: 从 API Gateway 到文件处理、感知并行（`asyncio.gather`），均实现非阻塞 IO。
- **SQLite 持久化**: 集成 SQLite 存储层，支持批次进度追踪与历史结果检索。
